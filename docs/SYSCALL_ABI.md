# INDOMINUS REX OS — System Call ABI

## Overview

Indominus Rex OS uses the x86-64 `syscall`/`sysret` (actually `iretq`) mechanism
for user→kernel transitions. The ABI follows Linux conventions where possible
to simplify future C/Rust userspace compatibility.

## Register Convention

| Register | Role |
|----------|------|
| RAX | Syscall number (in) / Return value (out) |
| RDI | arg0 |
| RSI | arg1 |
| RDX | arg2 |
| R10 | arg3 |
| R8 | arg4 |
| R9 | arg5 |

## Return Value Convention

- **Success**: RAX = 0 or positive result (e.g., byte count, PID)
- **Error**: RAX = negative errno (e.g., -1 = EPERM, -2 = ENOENT)

### Error Detection (userspace)

On x86-64, negative values in RAX appear as large unsigned values:

```c
// In C/Rust userspace:
long result = syscall(...);
if (result > -4096UL) {
    // Success (0 or positive)
} else {
    // Error: errno = -(int)result
}
```

Or equivalently:

```rust
let result = syscall(...) as i64;
if result >= -4095 {
    // Success
} else {
    let errno = -result;
    // Error
}
```

### Why This Works

The kernel returns values as signed 64-bit integers in RAX. A negative errno
like -14 (EFAULT) is stored as `0xFFFFFFFFFFFFFFF2` in RAX. When compared
as unsigned, this is greater than `0xFFFFFFFFFFFFF000` (-4096), making the
`> -4096UL` check reliable.

This is the **exact same convention** used by Linux, so musl and other C
libraries work unmodified.

## Syscall Numbers

| # | Name | Arguments | Return |
|---|------|-----------|--------|
| 0 | SYS_WRITE | fd, buf_ptr, count | bytes written or -errno |
| 1 | SYS_EXIT | exit_code | never (context switch) |
| 2 | SYS_YIELD | — | 0 |
| 3 | SYS_GETPID | — | current PID |
| 4 | SYS_WAITPID | child_pid | exit_code, 0 (running), or -errno |
| 5 | SYS_SLEEP | ticks (10ms) | 0 |
| 6 | SYS_READ | fd, buf_ptr, count | bytes read or -errno |
| 7 | SYS_PIPE | — | (read_fd << 32) | write_fd, or -errno |
| 8 | SYS_FORK | — | child PID (parent), 0 (child), or -errno |
| 9 | SYS_EXEC | path_ptr | -ENOSYS (not yet implemented) |
| 10 | SYS_CLOSE | fd | 0 or -errno |
| 11 | SYS_DUP | fd | new fd or -errno |

## File Descriptor Layout

| FD | Name | Description |
|----|------|-------------|
| 0 | stdin | Keyboard input (blocking read) |
| 1 | stdout | Serial output |
| 2 | stderr | Serial output |
| 3+ | — | Pipes, files (future) |

## Stack Layout

The kernel uses a 20-qword register frame saved on the kernel stack:

```
[rsp+0]   RAX  (syscall number / return value)
[rsp+8]   RBX
[rsp+16]  RCX  (user RIP, saved by CPU)
[rsp+24]  RDX  (arg2)
[rsp+32]  RSI  (arg1)
[rsp+40]  RDI  (arg0)
[rsp+48]  RBP
[rsp+56]  R8   (arg4)
[rsp+64]  R9   (arg5)
[rsp+72]  R10  (arg3)
[rsp+80]  R11  (user RFLAGS, saved by CPU)
[rsp+88]  R12
[rsp+96]  R13
[rsp+104] R14
[rsp+112] R15
[rsp+120] RIP  (IRET frame)
[rsp+128] CS
[rsp+136] RFLAGS
[rsp+144] RSP
[rsp+152] SS
```

## Security Invariants

1. **User pointer validation**: All user buffers are validated with
   `is_valid_user_range()` (bounds check) AND `is_user_buffer_mapped()`
   (full page table walk) before any kernel dereference.

2. **Return value convention**: Errors return negative errno, never
   `u64::MAX`. This prevents information leakage and matches Linux ABI.

3. **Process isolation**: Each process has its own PML4. User→kernel
   transition switches to kernel stack via per-CPU GSBase.

4. **No SYSRET**: Uses `iretq` instead of `sysretq` to avoid CVE-2012-0217.
